uuigaz.

uuigaz is a turn based game inspired by the guessing game of battleships. It is implemented in Java and Python using Protocol Buffers as a cross platform way of serializing protocol data into a compact and extensible format.

Rules

The game is played by two players, each trying to sink the enemy ships by guessing what position to fire at. Ships are laid out on a grid in secret by each player prior to a game and the first one to sink all enemy ships wins. In this variant of the game the following ship types and sizes are used:

  1. Submarine
  2. Destroyer
  3. Cruiser
  4. Battleship
  5. Carrier

Ships cannot share location and cannot be placed in a tile adjacent to another ship.

Login screen

Upon start each player is presented a login screen which can be used to choose which session to use, although no real authentication is done besides name.

Placement screen

On the placement screen ships are placed onto the board by clicking a grid and the keyboard arrow keys can be used to select orientation.

Game board

During the actual game phase the players own board are displayed on the top left and the opponent board down to the right. Misses are indicated by the teal color and hits as black. A sunken enemy ship is indicated by a log message as well as a morse SOS signal.

Test Client

There is also a text based test client written in Java that may double as a very stupid AI opponent. The test client will display boards and actions as unicode characters. In the above screenshot two test clients battle each other off, both randomly making moves until there is one winner.

Server and protocol

The server is written in Java and will upon broken connections remember the session. This means it is possible to make a move and quit the game while waiting for opponent to make his move. Communication with the server is done through messages that are serialized into the more or less standardized binary wire format which are then sent in a delimited form over TCP.

The messages are defined using an interface description language and are then compiled down to the language of choice, namely Python and Java in this project. Languages supported by the official compiler are:

  • C++
  • Python
  • Java

but there exist implementations for many other languages as well. A message could be defined as follows:

message Ident {
    required string name = 1;
    optional bool secure = 2;
}

and would compiled to Java result in a class with methods such as .getName(), .setName(String), .hasSecure() and .getSecure()

This has proven to be a very powerful and flexible concept that allow for large modifications even in late development without needing to rethink and reimplement a whole binary protocol. Implementing the protocol in different languages also becomes much easier as it is only the matter of executing a compiler. It is also quite efficient, for example the above definition would result in secure not being transferred at all unless explicitly set by the sender.

The name of the project, uuigaz, comes from runic inscriptions and seem to be old norse for 'warrior'. More on the subject can be found in the book "Texts and Contexts of the Oldest Runic Inscriptions", partly available and searchable on Google Books.
